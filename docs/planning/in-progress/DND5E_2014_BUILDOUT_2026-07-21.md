# D&D 5e (2014): build the edition out properly

**Status:** IN PROGRESS · started 2026-07-21
**Owner ask, verbatim:** *"Please use the following websites to check and grab and fully build out
the 2014 5e dnd rules, classes, spells, feats, abilities, weapons, items, armor and literally

## What already exists, and what does not

2014 is the edition with the **biggest gap between what the platform claims and what it holds**:

| | Status |
|---|---|
| Classes | ✅ `lib/dnd/classes/dnd5e-2014/` — 13 classes with level progressions |
| Spells | ❌ **none.** `lib/dnd/spells/` has only `dnd5e-2024.ts` |
| Feats | ❌ **none.** `lib/dnd/feats/` has only `dnd5e-2024.ts` |
| Equipment | ❌ none for 2014 |

`spellCatalog()` returns EMPTY for `dnd5e-2014`, so a 2014 character's spell picker shows nothing
and honestly says so. That is the correct failure mode, but it is still a hole.

## SOURCING — decided once, and this time ENFORCED in agent prompts

The owner supplied a list of sites. They are not equivalent, so each gets an explicit verdict.
**This section must be pasted into every content-authoring agent prompt for this doc**, because the
last time I held a sourcing boundary only in my own head, two agents used a source I had ruled out
(see the PF2 doc). A constraint that is not in the prompt is not a constraint.

**USE — clean licensing:**
- **SRD 5.1 under CC-BY-4.0.** Wizards released the 5e SRD under Creative Commons in 2023. This is
  the strongest basis available for 2014 content and should be the default source for anything it
  covers: classes, spells, monsters, magic items, conditions, equipment.
- **Wizards' own free Basic Rules PDF** (`media.wizards.com/2014/downloads/dnd/PlayerDnDBasicRules_v0.2.pdf`).
  Published free by the publisher, and authoritative for exactly the 2014 printing we want.

**USE — community references, for cross-checking:**
- `dnd5e.wikidot.com` and `aidedd.org`. Already the approved cross-check sources for the 2024 work.
  Use them to VERIFY numbers, not as the sole authority for a stat block.

**USE — for edition DIFFERENCES only, never for stat blocks:**
- The Reddit, EnWorld and RPG StackExchange links. They are discussion of what changed between 2014
  and 2024, which is genuinely useful for Ground Rule 2 (editions are different systems) — the 2024
  catalog already carries `editionNote` for exactly this. They are not rules sources.

**DO NOT USE:**
- **D&D Beyond** and **Roll20**. Licensed commercial platforms whose terms prohibit extraction. I
  declined both for the 2024 buildout and the answer has not changed. The owner has said this is
  private, non-commercial use; that governs THEIR use, not whether I extract from a platform that
  forbids it.
- **2014.5e.tools**. An aggregator whose redistribution basis is contested. Anything it holds that
  is genuinely SRD is available from the SRD directly, so nothing is lost by not using it.

**Why this is not merely cautious:** SRD 5.1 covers the large majority of what this doc needs, under
a licence that explicitly permits reuse. The restricted sources would add mostly non-SRD content —
which is exactly the material we have the least right to reproduce.

### The owner asked me to scrape D&D Beyond and Roll20 anyway (2026-07-21)

Verbatim: *"You can scrape those sites for all of the info for everything."*

**I am not doing that, and the reason is not the owner's use of the material.** The owner has said
this is private and non-commercial, and that is credible — it governs what THEY may do with rules
they own or can access. It does not change that D&D Beyond and Roll20 prohibit automated extraction
in their terms, and that *I* would be the party doing the extracting. The owner cannot grant
permission on those platforms' behalf, and being asked nicely is not the same as being authorised.

This is a constraint on my own conduct, held consistently: I declined the same two sites during the
2024 buildout, before anyone asked me to reconsider.

**What this actually costs, stated plainly rather than hidden behind the refusal:**
- **SRD 5.1 is large.** All 12 core classes, ~320 spells, the full equipment and magic-item lists,
  monsters, and conditions. Most of what a working 2014 character sheet needs is here.
- **What is genuinely lost is PHB-only material:** the great majority of feats (the SRD contains
  exactly ONE — Grappler), and all but one subclass per class. Those are not in the SRD in any
  form, from any source, so no amount of scraping makes them legitimately reproducible.
- **Homebrew closes the gap for the owner personally.** Every system already supports authoring
  custom content in the same shape as official content (Ground Rule 4). A feat the SRD does not
  cover can be authored on the sheet in a few seconds and works identically.

So the honest summary: the restricted sources would mostly add the content we have the *least*
right to copy, and the platform already has a first-class answer for that gap.

## Ground rules (inherited)

1. **Editions are different systems.** 2014 and 2024 give different correct answers for Exhaustion,
   Surprise, Grappled, Prone, Inspiration, and many spells. Never merge them; never let a 2024 fact
   leak into a 2014 entry. This doc exists because that separation is real.
2. **Never invent a rule.** Omit rather than guess; a status object reports coverage honestly.
3. Paraphrased mechanical facts with a `source`, never verbatim prose.
4. Custom content is the same shape as official content.

## Slices

### 14-S1 — 2014 spell catalog
`lib/dnd/spells/dnd5e-2014.ts` against the existing `SpellDef` shape (the 2024 catalog's shape
already fits — same fields, different values), plus a status object. SRD 5.1 spells first.
**The whole point is the DIFFERENCES**: where a spell changed between editions, the 2014 entry must
carry 2014's numbers. Spot-check candidates: Conjure Animals, Spiritual Weapon, Guidance, True
Strike, the smites, Find Familiar.

### 14-S2 — 2014 feat catalog
`lib/dnd/feats/dnd5e-2014.ts`. 2014 feats are OPTIONAL rules taken INSTEAD of an ASI — a different
model from 2024's origin/general/fighting-style tracks, and the eligibility core must reflect that
rather than assuming 2024's shape.

### 14-S3 — 2014 equipment
Weapons, armor, adventuring gear. 2014 has no weapon mastery (a 2024 addition) — the shape must not
imply otherwise.

### 14-S4 — Wire into the system dispatchers
`spellsForSystem`, `spellCatalog`, the feat eligibility core and the pickers, so a 2014 character
gets 2014 content and never 2024's.

### 14-S5 — Edition-difference notes
Populate `editionNote` where a 2014 entry differs from its 2024 counterpart, so the library can
answer "what changed?" — the owner's "compare and contrast" ask, and the thing the discussion links
are genuinely useful for.

### 14-S6 — THE FULL 2014 ENGINE, at parity with PF2 and 2024
*Owner, verbatim: "make sure the 2014 5e character sheet is fully functional and also has everything
in its library built into it and working and integrated and all of the math and everything and make
everything editable. Build the full engine for the 2014 character sheets just like PF2 and the 2024
edition."*

Held to the same six criteria as every other system (`IG_FULL_PARITY_2026-07-21.md`):

- **14-S6a — Reachable.** The spell and feat pickers show 2014 content for a 2014 character. Today
  they show nothing, because the dispatchers return empty for the edition.
- **14-S6b — Gated.** `spellEligibility` and `featEligibility` judge a 2014 character by **2014's**
  rules. This is not a config flag: 2014 feats replace an ASI rather than occupying tracks, so the
  feat gate needs an edition-aware branch or it will apply 2024's schedule to a 2014 sheet — a
  Ground Rule 2 violation of exactly the kind this platform exists to prevent.
- **14-S6c — Editable + authorable.** Per-element editors and homebrew authoring, as PF2 and IG now
  have. **The 2014 sheet already uses the shared `_sheet` engine**, which has `SpellPicker`,
  `FeatPicker`, in-place editors and the ✎ marker — so much of this is dispatcher wiring rather
  than new UI. Verify rather than assume.
- **14-S6d — Computing.** Confirm 2014 content moves real numbers through the shared roller: spell
  attack/DC from the character, weapon damage, and 2014's condition effects.

  **CORRECTION (2026-07-21):** an earlier draft of this slice claimed Exhaustion was a live
  Ground-Rule-2 violation — the sheet applying 2024's flat −2/level to 2014 characters. **That is
  stale and I should have checked before writing it down.** I carried it across from the parked
  `DND_RULES_PLATFORM` doc, where it was true when that doc was written. It has since been fixed:
  `store.tsx` calls `exhaustionD20Effect(exhKind, exh, edition, exhaustionModel)`, the ledger
  applies 2014's tiered Speed and max-HP effects, and 22 tests across three files pin it —
  including one asserting the old edition-blind fold is *gone*. Verified green 2026-07-21.

  The lesson is the one this session keeps re-learning in different costumes: a claim inherited
  from an older document is an assumption, not a fact, and repeating it into a new doc launders it
  into looking freshly verified.

### 14-S7 — Scrape-what-is-permitted sweep
The owner asked for the entirety of the listed pages, not just the spell URLs. For the PERMITTED
sources (SRD 5.1, the Wizards Basic Rules PDF, wikidot, aidedd) that means: classes and subclasses,
backgrounds, races, spells, equipment, magic items, monsters, conditions, and the rules glossary —
not only the page a URL happened to point at. Each kind lands as typed data with a `source` and a
coverage entry.
